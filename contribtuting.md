🤝 Contributing to CivicPulseThank you for your interest in contributing to CivicPulse! By contributing, you help transform unstructured public grievances across Hyderabad into actionable, data-driven insights for proactive local governance.Whether you want to optimize our LLM pipelines, fine-tune the predictive scoring algorithm, or build new UI features for the GHMC dashboard, we welcome your expertise.📌 Table of ContentsCode of Conduct[How Can I Contribute?](#-how-can- I-contribute)Development SetupArchitecture & Tech Stack ContextPull Request Guidelines🦺 Code of ConductBy participating in this project, you agree to maintain a respectful, inclusive, and collaborative environment. Treat all contributors with respect, deliver constructive feedback, and prioritize building a fair, unbiased tool for civic good.💡 How Can I Contribute?🐛 Reporting BugsCheck the existing issues list to ensure the bug hasn't already been reported.Open a new issue with a clear title, a description of the problem, steps to reproduce it, and your environment details (OS, Python version, etc.).✨ Feature RequestsSuggest new ideas by opening an issue. We are highly interested in:Additional localized data sources for Hyderabad (e.g., WhatsApp community channel parsers, other regional news feeds).Enhancements to the Multi-Agent scraping and geocoding accuracy.Improved spatial visualization capabilities for the GHMC zones.💻 Code ContributionsFork the repository.Create a localized feature branch (git checkout -b feature/amazing-civic-feature).Commit your changes with descriptive messages.Push to your branch and open a Pull Request (PR).🛠️ Development SetupTo get your local environment ready for active development:1. Clone the RepositoryBashgit clone https://github.com/your-username/CivicPulse.git
cd CivicPulse
2. Set Up a Virtual EnvironmentBashpython -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
3. Install DependenciesBashpip install -r requirements.txt
4. Configure Environment VariablesCreate a .env file in the root directory and append your specific API access tokens:Code snippetGEMINI_API_KEY=your_gemini_api_key_here
GROQ_API_KEY=your_groq_api_key_here
5. Launch the Local DashboardBashstreamlit run app.py
🏗️ Architecture & Tech Stack ContextBefore writing code, please review our existing architectural footprint to ensure your modifications integrate smoothly:Backend Processing: Built around Python, Pandas, crawl4ai, and langchain.AI Synthesis Layer: Employs Large Language Models (Gemini, Groq, or local Llama-3 variants via Ollama) to execute Named Entity Recognition (NER) for extracting categories, local Hyderabad landmarks, and language severity.Geocoding: Uses geopy to resolve raw text landmarks into dynamic geographic coordinates mapped strictly inside the 6 primary GHMC zones.The Impact Engine: Driven by a deterministic mathematical formula:$$Impact\ Score = (S \times 0.30) + (F \times 0.25) + (R \times 0.20) + (D \times 0.15) + (P \times 0.10)$$If modifying parameters, ensure the collective weights scale strictly to $1.0$.🚀 Pull Request GuidelinesTo maintain code health and project stability, ensure your PR adheres to these checkpoints:Code Formatting: Follow standard PEP 8 styles for Python code.Date & Traction Handling: Ensure any updates to the pipeline accurately maintain both the initial post_date and the dynamic traction_date properties to safeguard the dashboard's sorting integrity.UI Consistency: Any layout modifications to the frontend views must visually align with the minimalist aesthetic established in our Streamlit/HTML proof-of-concept.Testing: Run your scripts against the localized dataset (e.g., verifying landmark extraction for spots like Mehdipatnam, Kukatpally, or Tolichowki) to guarantee zero regressions.Questions?
